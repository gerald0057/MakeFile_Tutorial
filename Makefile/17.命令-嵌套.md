# 名词解释
把不同模块或是不同功能的源文件放在不同的目录中，在每个目录中都书写一个该目录的 mk，这有利于让 mk 变得更加简洁，而不至于把所有的东西全部写在一个 mk 中，增加可维护性

# 例子
```
submoduleA :
	cd submoduleA; make
```

等价于

```
submoduleA :
	make -C submoduleA
```

make 很多时候需要参数，定义变量 `MAKE`

```
MAKE = make -s

submoduleA :
	$(MAKE) -C submoduleA
```

# 变量传递
```
export var_a = value

var_b += val_1
var_b += val_2
export var_b
```

# 变量不传递
```
unexport variable
```
